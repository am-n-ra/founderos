# LEOS — Learning & Expertise Operating System
Last Updated: 2026-06-27

## Purpose

Bridge between what the founder needs to know (mission requirements) and what they currently know. Transforms capability gaps into structured learning paths.

## Core Pipeline

```
Ignorance → Knowledge → Skill → Competence → Expertise → Mastery
```

Minimizing: Random Learning, Tutorial Addiction, Information Overload, Skill Decay, Fake Expertise

## Prime Directive

Maximize Useful Expertise for mission success.

## Integration

- **MOS**: Learning priorities from mission needs
- **GROWTH.md**: Capability targets → learning paths
- **KNOWLEDGE.md**: Validated lessons stored here
- **DAILY_PLAN.md**: Time blocks for learning activities
- **SOS**: Energy management for sustained learning
- **CEOS**: Research for content topics
- **FAOS**: Market data for fundraising
- **PROFILE.md**: Current skill baseline

## Capability Gap Detection

1. Read MISSION.md → extract required capabilities per venture
2. Read PROFILE.md → extract current baseline
3. Read GROWTH.md → cross-check with existing targets and skill inventory
4. Compare → generate gap list
5. For each gap: propose learning path (resources, time estimate, priority)

### Gap Format

```
- Gap: <specific capability missing>
- Source: <mission/venture requirement>
- Current Level: <from PROFILE.md or GROWTH.md>
- Target Level: <intermediate/advanced>
- Path: <resource → project → validation>
- Time: <estimated hours>
- Priority: <HIGH/MEDIUM/LOW>
```

## Learning Formats

- **Active**: Deliberate research, experiments, projects, A/B testing
- **Passive**: Pattern detection from daily operations
- **Borrowed**: Learning from others (competitors, mentors, content)

## Workflow

1. **Identify gap** — from mission/priority mismatch
2. **Assess** — current level vs target
3. **Design** — approach (resource, project, or experiment)
4. **Execute** — with timebox
5. **Validate** — through application in real context
6. **Integrate** — into KNOWLEDGE.md with confidence level
