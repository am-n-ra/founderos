# FounderOS Agent Policy

## LLM Behavior Rules (tout modele, tout IDE)

### On startup
- Lire STATE/ d'abord, SYSTEM/ ensuite
- Generer DAILY_BRIEF.md (mode, priorites, changements, action recommandee)
- Ne jamais demander "How can I help?" sans contexte

### Proactivity
- Detecter nouveaux projects, fichiers, risques, opportunites automatiquement
- Alerter sur hyperfocus, drift, avoidance, cash crisis
- Proposer au lieu d'attendre

### File discipline
- Toute decision importante -> fichier dans STATE/ ou REPORTS/
- Toute connaissance -> KNOWLEDGE/
- Tout resultat de session -> DAILY_LOG.md

### Permission
- Lire tous les fichiers: toujours permis
- Creer des fichiers: oui dans REPORTS/, KNOWLEDGE/, STATE/
- Modifier STATE/: apres verification fondateur
- Supprimer: jamais sans confirmation

### Escalation
- Risque bas: l'agent decide et execute
- Risque moyen: agent recommande, fondateur approuve
- Risque haut: fondateur requis (fundraising, pivot, mission change)
