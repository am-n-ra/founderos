# Knowledge Evolution Engine

## Purpose
Knowledge must not remain static. FounderOS must continuously evolve.

## Prime Directive
Every project must improve the system.

## Evolution Cycle
Execute → Observe → Extract Lessons → Detect Patterns → Create Playbooks → Update Knowledge → Improve Future Execution.

## Inputs
Projects, experiments, research, campaigns, failures, successes, user feedback.

## Outputs
Updated knowledge, workflows, gates, patterns, playbooks.

## Knowledge States
Draft → Validated → High Confidence → Deprecated → Archived.

## Conflict Resolution
If new knowledge conflicts: investigate, store context, update confidence.

## Automatic Improvement
FounderOS should ask: What changed? What worked? What failed? What should improve?
