# Asset Registry

## Purpose
FounderOS must know every asset that exists. Never recreate blindly. Always search first.

## Asset Types
Entity, Environment, Action, Scene, Shot, Frame, Prompt, Audio, Video, Thumbnail, Script, Research.

## Registry Structure
ASSET_ID, TYPE, NAME, STATUS (DRAFT/ACTIVE/APPROVED/ARCHIVED/OBSOLETE), CREATED_DATE, PROJECT, DEPENDENCIES, USED_BY, VERSION, LOCATION.

## Dependency Tracking
Track: what created asset, what uses asset, what depends on asset.

## Reuse Rule
Before creating: search registry. If exists: reuse, update, version.

## Ultimate Objective
Complete production memory system. FounderOS can answer: where is this asset, who uses it, can it be reused, what depends on it.
