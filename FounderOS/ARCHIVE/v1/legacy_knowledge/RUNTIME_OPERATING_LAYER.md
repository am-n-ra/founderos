# Runtime Operating Layer

## Purpose
FounderOS must operate continuously, not only when prompted.

## Prime Directive
FounderOS is event-driven, not conversation-driven.

## Runtime Modules
- DAILY_OPERATIONS: daily briefings, daily reviews, task follow-ups
- SCHEDULER: time-based execution (08:00 brief, 20:00 review, weekly/monthly/quarterly)
- WATCHTOWER: monitor competitors, industry, technology, research, funding
- PROJECT_MONITOR: track progress, deadlines, blockers, dependencies, health (green/yellow/red)
- KNOWLEDGE_MONITOR: identify knowledge gaps, outdated info
- LEARNING_MONITOR: track skills, gaps, learning plans
- COMMUNICATION_MONITOR: track channels (email, calendar, slack, discord, WhatsApp)

## Alert Structure
ALERT_ID, CATEGORY, SEVERITY (Low/Medium/High/Critical), SUMMARY, IMPACT, RECOMMENDED ACTION.

## Continuous Loop
Observe → Analyze → Decide → Execute → Measure → Learn → Improve
