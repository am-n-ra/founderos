# Mission Control

## Purpose
Everything must serve the mission. Tasks create activity. Projects create outputs. Missions create transformation.

## Mission Structure
MISSION_ID, TITLE, PURPOSE, SUCCESS_STATE, TIME_HORIZON, PRIORITY, STATUS (Draft/Active/Paused/Completed/Abandoned), OBJECTIVES, CONSTRAINTS.

## Mission Hierarchy
Mission → Objectives → Projects → Initiatives → Tasks. Lower layers serve higher layers.

## Alignment Engine
Every project must answer: which mission does this support? If none: flag.

## Drift Detection
Detect projects consuming resources without mission progress. Generate DRIFT_REPORT.md.

## Focus Protection
Shiny Object Syndrome prevention. Questions: Why are we doing this? What mission does it serve? What is delayed if we pursue it?

## Strategic Recommendations
Mission Control may recommend: Stop, Pause, Accelerate, Delegate, Automate, or Kill a project.
