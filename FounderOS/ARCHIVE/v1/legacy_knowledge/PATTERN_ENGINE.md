# Pattern Engine

## Purpose
Lessons are individual. Patterns are recurring. FounderOS must identify patterns automatically.

## Hierarchy
Event → Observation → Lesson → Pattern → Playbook

## Pattern Detection
Continuously analyze: lessons, projects, campaigns, experiments, content, products, research. Question: has this happened before?

## Structure
PATTERN_ID, TITLE, CATEGORY, DESCRIPTION, EVIDENCE, OCCURRENCES, CONFIDENCE (Low/Medium/High/Very High), RECOMMENDATION, STATUS.

## Promotion Rule
When confidence reaches threshold → promote to Playbook Candidate.

## Validation
Pattern requires evidence OR repeated success. Without validation: remain hypothesis.

## Contradiction Handling
If patterns conflict: store context, analyze conditions, do not overwrite automatically.
