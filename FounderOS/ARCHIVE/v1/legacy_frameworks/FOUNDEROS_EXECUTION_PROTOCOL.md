# FOUNDEROS_EXECUTION_PROTOCOL V1

## Purpose
Bridge vision → execution → results.

## Prime Directive
Every objective must become: Project → Plan → Tasks → Deliverables → Outputs → Results.

## Golden Rule
No objective may remain abstract.

## Execution Pipeline
INPUT → ANALYSIS → PLAN → EXECUTION → VALIDATION → DOCUMENTATION → STATE UPDATE

## Stages

### Stage 1 — Mission Alignment
Does this contribute to the mission? Yes → continue. No → reject/delay/archive.

### Stage 2 — Objective Classification
Revenue, Product, Learning, Research, Hiring, Operations, Infrastructure, Health, Relationships, Admin.

### Stage 3 — Objective Decomposition
Large objectives are forbidden. Every objective must become executable.

### Stage 4 — Project Creation
Generate: PROJECT_PROFILE.md, PROJECT_STATE.md, PROJECT_ROADMAP.md, PROJECT_DECISIONS.md, EXECUTION_QUEUE.md.

### Stage 5 — Research Protocol
Research must answer: What do we know? What don't we know? What decision depends on this? Research without a decision is prohibited.

### Stage 6 — Decision Protocol
Generate DECISION_BRIEF.md with options, pros, cons, risks, recommendation.

### Stage 7 — Planning Protocol
Generate ACTION_PLAN.md with objectives, deliverables, dependencies, success criteria, timeline.

### Stage 8 — Execution Queue
All work enters EXECUTION_QUEUE.md. States: BACKLOG, READY, ACTIVE, BLOCKED, DONE, ARCHIVED.

### Stage 9 — Output Protocol
Every execution block must create output. No invisible work.

### Stage 10 — Validation Protocol
Verify: output exists, success criteria met, state updated, knowledge captured.

### Stage 11 — Documentation Protocol
Every significant result creates documentation.

### Stage 12 — State Update Protocol
Update: CURRENT_STATE.md, ACTIVE_PRIORITIES.md, PROJECT_STATE.md, MISSION_SCORE.md.

## Workflows by Objective

**Revenue:** REVENUE_PLAN.md, CUSTOMER_PROFILE.md, OFFER.md, CONTENT_PLAN.md, SALES_PIPELINE.md. Priority: Speed, Validation, Cash.

**Product:** PRODUCT_SPEC.md, FEATURE_BACKLOG.md, ROADMAP.md, TEST_PLAN.md. Priority: User Value, Validation, Iteration.

**Learning:** LEARNING_PLAN.md, SKILL_MAP.md, APPLICATION_PROJECT.md. Learning must connect to execution.

**Content:** CONTENT_STRATEGY.md, CONTENT_QUEUE.md, SCRIPT_LIBRARY.md, PUBLISHING_PLAN.md. Every asset must have: Goal, Audience, CTA, Success Metric.

**Research:** RESEARCH_REPORT.md, DECISION_BRIEF.md. Ends with recommendation.

## AI Workflow
When AI can execute, delegate. Founder focuses on: Judgment, Vision, Relationships, Sales, Critical Decisions.

## Escalation Policy
Escalate only when: Decision ambiguity, human relationship, legal risk, financial risk above threshold.

## Success Criteria
Outputs created, knowledge preserved, state updated, mission advanced.
