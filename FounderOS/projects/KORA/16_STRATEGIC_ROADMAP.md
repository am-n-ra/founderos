# KORA LAB — 16. STRATEGIC ROADMAP

**Version:** 1.0

---

# Purpose

The Strategic Roadmap translates the Master Plan into a sequence of executable strategic milestones.

The Master Plan defines the destination.

The Strategic Roadmap defines the path.

It identifies:

* what must happen first;
* what must wait;
* which milestones unlock the next stage;
* how today's execution contributes to a twenty-year mission.

The roadmap is reviewed regularly.

The Mission is not.

---

# Guiding Principle

Execution follows dependency,

not desire.

The next milestone is not necessarily the most exciting.

It is the one that unlocks the greatest number of future possibilities.

---

**Target Horizon:** Present → 12 months

### Objective

### 2. Compute Access

Secure sufficient compute through:

### 3. Sustainable Revenue

# Decision Filter

Before starting any initiative:

1. Which strategic horizon does this belong to?
2. Does it strengthen the current horizon?
3. Does it unlock future milestones?
4. Is it constrained by an earlier dependency?
5. Will this still matter five years from now?

If the initiative accelerates the current strategic horizon while preserving long-term optionality,

it belongs on the Roadmap.
