## Critical Capabilities

1. **Training Engineering** — Capacite a entrainer un modele from scratch sur compute limite. Decisif pour le premier modele.
2. **Data Engineering** — Collecter, nettoyer, structurer les donnees pour l'entrainement. Le bottleneck le plus sous-estime.
3. **Distribution** — Atteindre les utilisateurs. Sans ca, le meilleur modele est invisible.

## Capability Gap Analysis

- **Training:** On ne l'a jamais fait. Le premier modele sera une courbe d'apprentissage. Risque: prendre trop de temps.
- **Data:** Pas de pipeline existant. Commencer par scraper les sources publiques (Bible ewe, web, Masakhane).
- **Distribution:** Zero. Commencer par le cercle proche (devs, etudiants).
