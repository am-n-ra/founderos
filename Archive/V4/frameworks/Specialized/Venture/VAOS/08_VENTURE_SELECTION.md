# VAOS — 08. VENTURE SELECTION FRAMEWORK

**Version:** 1.0
**Phase:** III — Strategic Architecture
**Owner:** Founder
**Status:** Operational (Reviewed annually or per new venture proposal)

---

# Purpose

The Venture Selection Framework defines how new ventures enter the Founder Ecosystem.

Ideas are unlimited.

Time, capital and attention are not.

The objective is therefore **not to create many companies.**

The objective is to create only ventures that increase the probability of accomplishing the Life Mission.

Every proposed venture must pass this framework before resources are committed.

---

# Core Principle

A venture is not created because it is profitable.

A venture is not created because it is interesting.

A venture exists only if it removes a structural bottleneck that prevents the ecosystem from accomplishing its mission.

Every venture must create capabilities.

Never distraction.

---

# The Founder Ecosystem

The ecosystem should behave like a living organism.

Each venture has a role.

No venture exists independently.

Each strengthens the others.

The ecosystem is therefore evaluated as a whole.

Not company by company.

# Venture Entry Conditions

Every proposed venture must satisfy all mandatory conditions.

---

## Condition 1

Mission Alignment

Does this directly advance the Life Mission?

If No,

Reject.

---

## Condition 2

Structural Problem

Does it solve a structural bottleneck?

Or merely a temporary inconvenience?

Temporary problems rarely deserve companies.

---

## Condition 3

Capability Creation

Does this create new capabilities?

Or merely redistribute existing value?

Preference always goes to capability creators.

---

## Condition 4

Compounding

Will its value increase over years?

Or disappear after one project?

Compound systems dominate linear systems.

---

## Condition 5

Ecosystem Synergy

Does it strengthen existing ventures?

Or compete with them?

Internal competition destroys ecosystems.

---

## Condition 6

Strategic Independence

Does it increase technological sovereignty?

Or increase dependency?

Dependency is a strategic cost.

---

## Condition 7

Founder Advantage

Do we possess a unique advantage?

Examples:

* knowledge;
* timing;
* data;
* distribution;
* engineering;
* language;
* geography;
* relationships.

Without unfair advantage,

competition becomes commodity.

---

## Condition 8

Execution Feasibility

Can this realistically be built with available resources?

Impossible ideas are postponed.

Not rejected.

# Opportunity Cost

Creating one company means not creating another.

Therefore every new venture asks:

What higher-leverage opportunity are we delaying?

If opportunity cost is greater than expected value,

the venture waits.

---

# Exit Criteria

A venture should be redesigned, merged or terminated if:

* it no longer advances the Life Mission;
* another venture performs its role better;
* the bottleneck disappears;
* the ecosystem changes;
* resources create greater leverage elsewhere.

Companies serve the mission.

The mission never serves companies.

# Portfolio Balance

The ecosystem should remain balanced.

Too many research ventures create no revenue.

Too many revenue ventures stop innovation.

Too many experimental ventures create chaos.

The portfolio should continuously balance:

* Research
* Infrastructure
* Revenue
* Accessibility
* Multiplication

---

# Decision Filter

Before creating a new venture, ask:

1. Which ecosystem bottleneck disappears if this exists?
2. Which capability does it create?
3. Which existing ventures become stronger?
4. Would the ecosystem be weaker without it?
5. Is a new company actually necessary?

If the answer to Question 5 is **No**,

do not create a company.

Expand an existing venture instead.
