# PMOS — Project Management Operating System
Last Updated: 2026-06-27

## Purpose
Structured project lifecycle management. Tracks each project through IDEA → VALIDATION → LAUNCH → GROWTH → SCALE → MATURE.

## Project Lifecycle
- **IDEA**: Raw concept, 1-pager in project README
- **VALIDATION**: Market research, customer interviews, prototype
- **LAUNCH**: MVP live, first users/customers
- **GROWTH**: Scaling, team building
- **SCALE**: Multiple revenue streams
- **MATURE**: Sustainable, maintenance mode

## State per Project
Each project tracked in PRIORITY_MATRIX.md with:
- Phase, Priority, Warning/Blocker
- Last action date
- Next milestone

## Integration
- **DAOS**: Daily execution tasks per project
- **TIMELINE**: Milestone events logged
- **PROJECTS_MASTER_INDEX**: Central registry

## Project Health Check
For any project, check:
1. Is README.md current?
2. Is KNOWLEDGE.md updated with recent learnings?
3. Are blockers documented?
4. Is next milestone defined?
5. Are accounts current?

## Core Principles (from v2)
- Every project serves mission or survival
- Portfolio optimization over individual project preference
- Max 3 mission-critical, max 5 active projects
- Measure by milestones, not activity
- If blocked > 7 days → escalate
- Kill early if evidence suggests failure
