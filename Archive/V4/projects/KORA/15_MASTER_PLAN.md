# KORA LAB — 15. MASTER PLAN

**Version:** 1.0

---

# Purpose

The Master Plan defines the long-term transformation path of Kora Lab.

It answers one fundamental question:

**In what order must Kora build its capabilities so that the mission becomes inevitable?**

Every organization has limited resources.

The sequence of execution therefore matters more than the ambition itself.

The Master Plan establishes the irreversible order in which Kora should evolve.

---

# Guiding Principle

Kora does not optimize for speed.

Kora optimizes for permanence.

Every phase should create the conditions that make the next phase possible.

Each phase should permanently strengthen the institution.

Nothing should be built simply because it is fashionable.

Everything should contribute to the Mission.

---

# Phase I

## Objective

Ensure that Kora survives long enough to begin building sovereign AI.

Without survival,

there is no research.

Without research,

there is no sovereignty.

# Transformation Logic

Every phase creates the conditions for the next.

```text
Survival
      ↓

Capability
      ↓

Adoption
      ↓

Ecosystem
      ↓

Institution
```

Skipping a phase weakens everything that follows.

---

# Resource Allocation Philosophy

During each phase,

approximately 70% of resources should strengthen the current phase,

20% should prepare the next,

10% should explore future opportunities.

This balance prevents both stagnation and premature expansion.

---

# Strategic Constraints

Kora should never optimize for later phases before mastering earlier ones.

Examples:

Do not prioritize global branding before product quality.

Do not prioritize ecosystem building before technological capability.

Do not prioritize institutional governance before organizational stability.

The sequence is strategic.

---

# Continuous Principles

Across all phases,

the following remain constant:

* Mission before valuation.
* Sovereignty before convenience.
* Excellence before compromise.
* Accessibility alongside performance.
* Long-term capability over short-term visibility.
* Integrity over acceleration.

---

# Phase Transition Criteria

A phase is completed when its core capabilities become repeatable,

not when isolated successes occur.

Progression is based on organizational maturity,

not calendar time.

---

# Decision Filter

Before approving any major initiative:

1. Which phase are we currently in?
2. Does this initiative strengthen the objectives of this phase?
3. Does it prepare the next phase?
4. Are we trying to solve problems belonging to a future phase prematurely?
5. Does this decision increase the probability that the Master Plan succeeds?

If the initiative primarily serves a future phase while the current phase remains incomplete,

it should usually be postponed.
